
#---------------------------------------------------------------------------
#  UIElements Expansions
#---------------------------------------------------------------------------

Unity UIElements Expansions is an asset series that extends Unity's UIElements.
This asset primarily adds the following features to the UnityEditor:

1. Global Style Sheet
The CSS that the USS file is based on is basically used by writing something common in the header of the web page.
“Global Style Sheet” is a function that can use USS as a common style file.

2. Inspector design automatic assignment function
Before UIElements appeared, you had to write an editor script to change the design and placement of the inspector.
With the advent of UXML, you can now write these designs as if you were writing HTML.
However, in order to apply this and style sheets, it is necessary to create a short editor script.
This asset eliminates the hassle.
If you place a uxml file with the same name as the class you want to apply in the Assets folder (anywhere), the uxml file is automatically applied to the Inspector.
Furthermore, if you place a uss file with the same name as the class to be applied to the Assets folder, the uss file can be applied only to the inspector of that component.
In addition to components that inherit MonoBehaviour and ScriptableObject, you can also create your own classes with the [System.Serializable] attribute.

3. Additional Tags / Features / Class
We've added some useful tags to the UIElements UXML tag.
Please see the following page for details.

Unity UIElements ExpansionsはUnityのUIElementsを拡張するアセットシリーズです。
このアセットでは主に下記のような機能がUnityEditorに追加されます:

1.グローバルスタイルシート
USSファイルの元となっているCSSは、基本的にWebページのヘッダーに共通するものを記述することによって使用されます。
「グローバルスタイルシート」は、共通のスタイルファイルとしてUSSを利用できる機能です。

2.インスペクターデザイン自動アサイン機能
UIElements登場以前は、インスペクターのデザインや配置を変更するときにエディタースクリプトを記述する必要がありました。
UXML登場で、HTMLを記述するような感覚でこれらのデザインを記述することができるようになりました。
しかし、これやスタイルシートを適用するためには短いながらもエディタースクリプトを作成する必要があります。
このアセットはその面倒臭さを無くします。
適用したいクラスと同じ名前のuxmlファイルをAssetsフォルダー（どこでも）に配置すると、uxmlファイルが自動的にインスペクターに適用されます。
さらにAssetsフォルダーに適用するクラスと同じ名前のussファイルを配置すると、そのコンポーネントのインスペクターにのみussファイルを適用できます。
MonoBehaviourとScriptableObjectを継承するコンポーネントに加えて、[System.Serializable]属性を持つ独自のクラスに関しても作成することができます。

3.追加タグ / 機能 / クラス
UIElementsのUXMLに記載するタグに便利な機能をもったタグをいくつか追加しています。

Unity UIElements Expansions是一个扩展Unity UIElements的资产系列。
此资产主要向UnityEditor添加以下功能:

1.全局样式表
作为USS文件的来源的CSS基本上用于描述在网页报头的什么普通。
「全局样式表」是一种可以将USS用作通用样式文件的功能。

2.检查器设计自动指派功能
在UIElements出现之前,您必须在更改检查器的设计和放置时编写编辑器脚本。
随着UXML的出现,现在可以像描述HTML一样描述这些设计。
但是,为了应用此样式和样式表,您需要创建一个简短的编辑器脚本。
这种资产消除了它的麻烦。
如果将与您要应用的类同名的uxml文件放在Assets文件夹中(无论哪里),uxml文件将自动应用于检查器。
此外,如果放置与您应用于Assets文件夹的类同名的uss文件,则您只能将uss文件应用于该组件的检查器。
除了继承MonoBehaviour和ScriptableObject的组件之外,您还可以创建具有[System.Serializable]属性的自己的类。

3.添加标签/功能/类别
我们在UIElements的UXML中添加了一些具有有用功能的标签。

#---------------------------------------------------------------------------
#  UIElements Expansions
#---------------------------------------------------------------------------

Unity UIElements Expansions는 Unity의 UIElements를 확장하는 자산 시리즈입니다.
이 자산은 주로 다음과 같은 기능이 UnityEditor에 추가됩니다:

1. 글로벌 스타일 시트
USS 파일의 원본이있는 CSS는 기본적으로 Web 페이지의 헤더에 공통적를 작성하여 사용됩니다.
"글로벌 스타일 시트"는 일반적인 스타일 파일로 USS를 사용할 수있는 기능입니다.

2. 관리자 디자인 자동 할당 기능
UIElements 등장 이전에는 관리자의 디자인과 배치를 변경할 때 편집기 스크립트를 작성해야했습니다.
UXML 등장으로 HTML을 작성하는 것 같은 감각으로이 디자인을 설명 할 수있게되었습니다.
그러나 이것은 나 스타일 시트를 적용하기 위해서는 짧지 만 편집기 스크립트를 작성해야합니다.
이 자산은 그 귀찮은 로움을 없애고 있습니다.
적용 할 클래스와 동일한 이름의 uxml 파일을 Assets 폴더 (어디서나)에 배치하면 uxml 파일이 자동으로 관리자에 적용됩니다.
또한 Assets 폴더에 적용하는 클래스와 동일한 이름의 uss 파일을 배치하면 해당 구성 요소 검사기에만 uss 파일을 적용 할 수 있습니다.
MonoBehaviour과 ScriptableObject을 상속하는 구성 요소뿐만 아니라, [System.Serializable] 속성을 가진 자신의 클래스에 관해서도 만들 수 있습니다.

3. 추가 태그 / 기능 / 클래스
UIElements의 UXML에 기재 태그에 유용한 기능을 가진 몇 가지 태그를 추가하고 있습니다.
